# commit.md

## 目的
コミット履歴を統一し、変更内容を明確にする

## 形式

<type>: <日本語要約>

例：
feat: ログイン機能を追加
fix: 保存処理の不具合を修正
refactor: API処理を分割

## type

- feat: 新機能
- fix: バグ修正
- refactor: リファクタ
- docs: ドキュメント
- chore: 設定・雑務
- style: フォーマットのみ
- perf: パフォーマンス改善
- test: テスト

## ルール
- typeは英語
- 本文は日本語
- 1行で書く
- 主語は省略
- 「〜を追加」「〜を修正」で書く

## 禁止
- 曖昧な表現（例：変更、対応）
- 複数内容を1コミットに含める

## 原則
- 1コミット1責務
